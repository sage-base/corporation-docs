# 地方議会の仕組み

## 統一地方選挙と議員任期

地方議会の議員は、ほとんどが**4年ごとの統一地方選挙**のタイミングで改選されます。

### 統一地方選挙とは

統一地方選挙は、全国の地方自治体の選挙を同じ時期にまとめて行う制度です。

- **実施時期**: 4年に1度（4月）
- **対象**: 都道府県議会議員、市区町村議会議員、知事、市区町村長
- **直近の実施**: 2023年4月（次回は2027年4月）

### メンバー構成の特徴

統一地方選挙により、**地方議会のメンバーは4年間大枠変わりません**。

```
第20期（2019年〜2023年）→ 統一地方選挙 → 第21期（2023年〜2027年）
```

途中で変更が発生するケース：

- 議員の辞職・死亡
- 補欠選挙による新議員の就任
- 会派の移動・分裂・結成

## 期（term）の管理

地方議会では「期」という単位でメンバーを管理します。

| 用語 | 説明 | 例 |
|------|------|-----|
| 期 | 選挙から次の選挙までの4年間 | 第21期 |
| 年度 | 会計年度（4月〜翌3月） | 令和5年度 |
| 定例会 | 年数回開催される定例の議会 | 令和5年第3回定例会 |

## Sagebaseでの表現

### Electionエンティティ

選挙をファーストクラスのエンティティとして管理することで、「いつ行われた選挙で決まった期か」が明確になります。

```python
Election(
    governing_body_id=1,      # 京都市
    term_number=21,           # 第21期
    election_date=date(2023, 4, 9),  # 選挙実施日
    election_type="統一地方選挙"
)
```

### Conferenceエンティティ

会議体には`term`と`election_id`を設定します。

```python
Conference(
    name="議会",
    governing_body_id=1,      # 京都市
    term="第21期",
    election_id=1             # 2023年統一地方選挙
)
```

## 統一地方選挙に参加していない自治体

一部の自治体は、合併や首長の辞職などの理由で統一地方選挙とは異なるタイミングで選挙を行っています。この場合も`Election`エンティティで個別に管理できます。
