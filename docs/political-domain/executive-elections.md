# 首長選挙

## 議会議員とは異なる選挙サイクル

知事や市長などの首長選挙は、**統一地方選挙とは別のタイミングで行われることが多い**です。

### 理由

- 首長の辞職・死亡による補欠選挙
- 不信任決議による失職
- 合併に伴う選挙
- 歴史的な経緯（統一地方選挙制度開始前からのずれ）

### 具体例

| 自治体 | 議会議員選挙 | 首長選挙 |
|--------|-------------|----------|
| 東京都 | 統一地方選挙（4月） | 7月頃（都知事選） |
| 大阪市 | 統一地方選挙（4月） | 11月頃（市長選） |
| 京都市 | 統一地方選挙（4月） | 2月頃（市長選） |

## Sagebaseでの管理方針

首長と議会議員は**別のサイクルで管理する**ことを推奨します。

### 理由

1. **任期開始日が異なる**: 議員は統一地方選挙後、首長は個別の選挙後
2. **データの独立性**: 議会メンバーの変更と首長の変更は連動しない
3. **追跡の容易さ**: 首長の交代を個別に記録できる

### データモデル上の考慮点

現在のSagebaseでは、首長は`Politician`エンティティとして管理され、議会（`Conference`）への所属とは別に扱われます。

```
議会議員 → Conference に所属（PoliticianAffiliation）
首長     → GoverningBody の代表として別途管理（将来的な拡張）
```

## 今後の拡張可能性

首長専用のエンティティやリレーションを追加することで、より詳細な追跡が可能になります。

- 首長の任期管理
- 首長選挙の結果記録
- 議会との関係（与党・野党構成など）
